## 2. 💡 핵심 알고리즘 & 접근 아이디어
* **Product of Array Except Self (나외의 모든 곱):**
    * 나눗셈을 못 쓰는 제약 조건이 걸렸을 때는 **'나를 기준으로 왼쪽 모든 곱' ❌ '나를 기준으로 오른쪽 모든 곱'**으로 나누어 정답을 조립하는 **누적 곱(Prefix & Suffix Product)** 방식을 떠올려야 함.
* **Valid Sudoku (유효성 검사):**
    * 굳이 모든 칸을 n제곱으로 뒤지는 게 아니라, 행/열/3x3 박스별로 `HashSet`을 두어 **한 번 본 숫자를 기억**하고 한 바퀴만 쓱 돌면서 중복을 체크하면 무식해 보여도 $O(1)$(크기가 9x9로 고정이라)로 풀림.

## 3. ☕ 언어 문법 & 자료구조 (Java)

* **String의 불변성과 정렬법:**
    * **핵심:** Java `String`은 `sort()`가 없음. `char[]` 복사본을 만들어서 정렬한 뒤 새 문자열을 생성해야 하며, 이때 원본은 그대로 유지됨.
    * **코드 예시:**
      ```java
      char[] c = str.toCharArray();
      Arrays.sort(c);
      String sortedStr = new String(c);
      ```

* **Map.Entry와 entrySet():**
    * **핵심:** `Entry`는 키-값 한 쌍의 껍데기 객체. `map.entrySet()`을 호출하면 루프를 돌릴 수 있는 주머니(Set)로 반환되어 2차원 배열 변환 등 키와 값을 동시에 조작할 때 유용함.
    * **코드 예시:**
      ```java
      for (Map.Entry<Integer, Integer> entry : map.entrySet()) {
          int key = entry.getKey();
          int val = entry.getValue();
      }
      ```

* **2차원 배열 정렬 공식 (`a - b` vs `b - a`):**
    * **핵심:** 정렬 방향이 헷갈릴 땐 최종 목표의 알파벳 순서를 생각할 것! **오름차순은 `a - b`**, **내림차순은 `b - a`**. 단순 뺄셈은 대단히 큰 수 비교 시 오버플로우가 나므로 `Integer.compare()`가 안전함.
    * **코드 예시:**
      ```java
      // 오름차순 (a-b)
      Arrays.sort(arr, (a, b) -> Integer.compare(a[0], b[0]));
      // 내림차순 (b-a)
      Arrays.sort(arr, (a, b) -> Integer.compare(b[0], a[0]));
      ```
